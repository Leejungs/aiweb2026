SECTION 4 · STEP 2 보강

git push 인증 — Personal Access Token

2021년부터 GitHub은 비밀번호로 HTTPS push 불가. PAT(Personal Access Token)로 비밀번호를 대체한다. 두 종류 중 수업에서는 fine-grained 사용 (리포 단위 권한 + 만료일 강제).

종류장점단점
Tokens (classic)scope가 큼 (모든 리포 일괄)리포 단위 분리 불가, 만료 무한 가능
Fine-grained tokens ⭐리포 단위 권한 + 만료일 강제설정 항목이 많음 (이번 슬라이드에서 정리)

🔑 PAT 생성 (1) — 발급 페이지 진입

github.com/settings/tokens?type=beta

  • 1 우상단 프로필 아이콘 → Settings
  • 2 좌측 메뉴 맨 아래 → Developer settings
  • 3 Personal access tokens → Fine-grained tokens
  • 4 Generate new token 클릭

🔑 PAT 생성 (2) — 이름 + 만료일

Token name: aiweb2026-push

  • 1 Token name: aiweb2026-push (식별 가능한 이름이면 OK)
  • 2 Expiration: 90 days 권장 (학기 끝까지 — 무한 만료 비추천)
  • 3 Description: 자유 (예: aiweb2026 학기 push용)

🔑 PAT 생성 (3) — 리포지토리 접근 범위

Repository access: All repositories

옵션의미수업 권장
Public repositories본인 public 리포 read-only❌ push 안 됨
All repositories본인 소유 모든 리포 (현재 + 미래)⭐ 학기 동안 편리
Only select repositories지정한 리포만 (최대 50개)보안 우선이면 선택

리포 이름이 aiweb2026처럼 1개여도 다음 주 CI/CD에서 새 리포가 생길 수 있으니 All repositories가 운영 부담 적음.

🔑 PAT 생성 (4) — 권한 설정 + 토큰 발급

Repository permissions: Contents R/W · Metadata R · Workflows R/W

권한설정이유
ContentsRead and writegit push 핵심 권한
MetadataRead-only (자동 필수)리포 정보 조회 — 끌 수 없음
WorkflowsRead and write10주차 GitHub Actions 워크플로 push에 필요
중요

Generate token 클릭 → 토큰이 화면에 1회 노출 → 즉시 복사. 화면 닫으면 다시 못 봄 → 새로 발급해야 함.

🔧 STEP 2.6 — git remote set-url로 토큰 적용

발급받은 토큰을 origin URL에 박아두면 push할 때마다 비밀번호를 묻지 않는다.

# 1. 현재 origin URL 확인
git remote -v
# origin  https://github.com/<username>/<repo>.git (fetch)
# origin  https://github.com/<username>/<repo>.git (push)

# 2. HTTPS URL에 토큰 임베딩 (set-url)
git remote set-url origin https://<username>:<TOKEN>@github.com/<username>/<repo>.git

# 3. 변경 확인 (토큰이 그대로 보이면 정상)
git remote -v

# 4. push — 이제 비밀번호 묻지 않음
git push origin main
보안

<TOKEN>은 공개 코드 / 슬랙 / Discord / 스크린샷에 절대 노출 금지. 실수로 push되면 즉시 GitHub Settings → Tokens → Revoke.
