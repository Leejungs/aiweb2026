2SSH 키 만들기 (본인 PC, 약 1분)

인스턴스에 접속할 키 한 쌍을 본인 노트북에서 미리 만든다. 공개키(.pub)는 STEP 3에서 붙여넣고, 개인키는 본인만 보관한다.

macOS / Linux / WSL

mkdir -p ~/.ssh
ssh-keygen -t ed25519 -f ~/.ssh/oracle_key -N ""
cat ~/.ssh/oracle_key.pub   # → 출력된 한 줄을 복사

Windows PowerShell

mkdir $env:USERPROFILE\.ssh -ErrorAction SilentlyContinue
ssh-keygen -t ed25519 -f $env:USERPROFILE\.ssh\oracle_key -N '""'
Get-Content $env:USERPROFILE\.ssh\oracle_key.pub | clip
# → 클립보드에 공개키 복사 완료
왜 미리? Oracle 콘솔에서 "키 다운로드" 옵션도 있지만, 본인이 만든 키를 붙여넣는 쪽이 더 안전하고 분실 시 재발급도 쉽다.

3VM 인스턴스 생성

3-1. 진입

좌측 햄버거 메뉴 → Compute → Instances 클릭.

햄버거 메뉴 → Compute → Instances

Instances 목록 화면에서 Create instance 버튼 클릭.

3-2. Basic information (Name)

원하는 이름 입력. 예: aiweb2026-demo. Compartment는 기본값(루트), Availability domain도 기본값(AD-1) 그대로 둔다.

Name + Compartment + Availability domain (기본값)

3-3. Image

Change image → Canonical Ubuntu 22.04 선택 → Select image.

Canonical Ubuntu 22.04 (Free, Shielded instance)

3-4. Shape — 가장 중요

  1. Change shape 클릭
  2. Shape series 탭에서 Specialty and previous generation 선택
  3. VM.Standard.E2.1.Micro 라디오 버튼 선택 (1 OCPU · 1 GB · Always Free-eligible 라벨 확인)
  4. Select shape 클릭

Specialty and previous generation → VM.Standard.E2.1.Micro (Always Free-eligible)

다시 강조 — Ampere(A1.Flex)는 Seoul/Chuncheon에서 capacity가 거의 항상 부족하다. 9주차는 E2.1.Micro 한 가지로 진행한다. AMD/Intel 탭의 E3/E4/E5.Flex는 유료.

3-5. Security

건드릴 것 없음. Shielded instance 토글 OFF(기본값) 그대로 두고 Next.

Security: 기본값 그대로 통과

3-6. Networking

  • Primary network: Create new virtual cloud network
  • VCN name / Subnet name: 자동 생성된 이름 그대로 (예: vcn-20260429-1700)
  • Subnet: Create new public subnet (Private 아님)
  • CIDR block: 기본값 10.0.0.0/24
  • Private IPv4 address: Automatically assign private IPv4 address
  • Public IPv4 address: ON (자동 할당)

Primary VNIC: Create new virtual cloud network

Public subnet 자동 생성 (CIDR 10.0.0.0/24)

Private IPv4: Automatically assign

Public IP 토글이 안 켜지면 일단 OFF로 두고 진행 → STEP 4 하단의 "Public IP 수동 추가" 참조.

3-7. SSH Keys

  • Paste public key 라디오 버튼 선택
  • STEP 2에서 복사한 공개키 한 줄 붙여넣기 (ssh-ed25519 AAAA...로 시작)

Paste public key → STEP 2 출력 한 줄 붙여넣기

3-8. Boot Volume (Storage)

  • Specify a custom boot volume size 토글 ON
  • Boot volume size: 50 GB (기본 권장. Always Free 한도 내에서 200까지 가능하지만 늘리기는 가능한데 줄이기가 불가능. 50GB로 만들자.)
  • VPU: 10 (Balanced, 기본값)

Boot volume 50 GB · Balanced (3000 IOPS / 24 MB/s)

3-9. Create

화면 하단 Create 클릭. State가 Provisioning → 1~2분 뒤 Running으로 바뀐다.

생성 직후: Provisioning + Always Free 라벨

1~2분 뒤 Running으로 전환

Running이 떴는가? 인스턴스 이름을 클릭해 상세 페이지의 Public IPv4 Address가 보이면 STEP 3 끝.

4Public IP 확인 + SSH 접속 테스트

4-1. Public IP 메모

  1. Compute → Instances → 본인 인스턴스 클릭
  2. 상세 화면 Instance access 섹션의 Public IP address 값 옆 Copy 버튼 (예: 158.179.195.124)
  3. 아래 Username은 ubuntu로 표시됨 — SSH 접속 시 이 사용자명으로 들어간다

Instance details → Instance access → Public IP address / Username

4-2. SSH 접속 (본인 노트북에서)

# macOS/Linux/WSL
chmod 600 ~/.ssh/oracle_key
ssh -i ~/.ssh/oracle_key ubuntu@<PUBLIC_IP>
# 첫 접속 시 fingerprint → yes

프롬프트가 ubuntu@instance-...:~$로 바뀌고 whoami 결과가 ubuntu로 나오면 성공.

4-3. (Public IP 토글이 안 켜졌을 때만) 수동 추가

  1. 인스턴스 상세 → 상단 Networking 탭
  2. Quick actions → Connect public subnet to internet → Connect 버튼
  3. 그래도 IP가 없으면: Networking 탭의 Attached VNICs → VNIC 클릭 → IP administration → Private IP 행 ⋮ → Edit → Public IP type: Ephemeral Public IP → Update

5(권장) 예산 알림 설정 — 1분

Always Free 한도 안에서는 0원이지만, 실수로 유료 자원을 만들었을 때 즉시 메일이 오게 안전장치를 건다.

  1. 좌측 햄버거 메뉴 → Billing & Cost Management → Budgets
  2. Create Budget
  3. Target type: Compartment → 루트 compartment 선택
  4. Budget amount: 1 (USD)
  5. Alert rule: Forecast / Actual = 50% → 본인 이메일 입력
  6. Save

햄버거 메뉴 → Billing & Cost Management → Budgets

강사가 제출된 IP를 일괄로 Cloudflare DNS A 레코드(<id>-demo.aiweb2026.site)로 등록한다. 학생은 시트 제출까지만.

7완료 체크리스트

  • Oracle Cloud 계정 생성 + 홈 리전 한국으로 설정
  • SSH 키 한 쌍 본인 PC에 보관 (~/.ssh/oracle_key, oracle_key.pub)

  • Public IPv4 Address 발급됨
  • 본인 노트북에서 ssh -i ... ubuntu@<IP> 접속 성공
  • 예산 알림($1) 등록 (권장)
  • 분배 시트에 본인 ID + Public IP 제출
